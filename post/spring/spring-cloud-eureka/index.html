Spring Cloud Config 配置服务

Posted @ 2017-09-08 · 1191 Words · Tags: springcloud · Hits:

spring cloud 配置服务提供了一种非常方便的应用配置管理功能，用来解决分布式系统中配置文件太多导致的各种问题，而它自身的配置也非常简单。

服务端用来从git获取配置信息，并提供ws接口供客户端调用。客户端在启动初将从指定的服务端获取配置信息，后续也可刷新配置。

spring cloud 配置服务同时提供了git和svn两种方式，下面是一个使用git的示例。

服务端：

1.在github上新建仓库，名称config-repo。创建3个配置文件config-dev.properties，config-test.properties，config-pro.properties。内容分别如下：

id=dev
desc=dev
db=1000

id=pro
desc=pro
db=2

id=test
desc=
db=90

2.新建quickstart Maven项目，修改pom文件如下：

<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>1.5.2.RELEASE</version>
    </parent>
    
    <modelVersion>4.0.0</modelVersion>
    <artifactId>config</artifactId>
    <packaging>jar</packaging>
    <name>config</name>

    <properties>
        <java.version>1.8</java.version>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
    </properties>

    <dependencyManagement>
            <dependencies>
                <dependency>
                    <groupId>org.springframework.cloud</groupId>
                    <artifactId>spring-cloud-dependencies</artifactId>
                    <version>Dalston.SR1</version>
                    <type>pom</type>
                    <scope>import</scope>
                </dependency>
            </dependencies>
        </dependencyManagement>
        <dependencies>
            <dependency>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-starter-web</artifactId>
            </dependency>
            <dependency>
                <groupId>org.springframework.cloud</groupId>
                <artifactId>spring-cloud-config-server</artifactId>
            </dependency>
        </dependencies>

    <build>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>

    <repositories>
        <repository>
            <id>spring-releases</id>
            <url>https://repo.spring.io/libs-release</url>
        </repository>
    </repositories>
    <pluginRepositories>
        <pluginRepository>
            <id>spring-releases</id>
            <url>https://repo.spring.io/libs-release</url>
        </pluginRepository>
    </pluginRepositories>
</project>

3.在resource中添加配置文件application.yml，内容如下：

server:
  port: 8088
spring:
  application:
    name: config
  cloud:
    config:
      server:
        git:
          uri: https://github.com/luangeng/config-repo/
          search-paths: config-repo
          username:
          password:

4.添加Application类

package com.luangeng;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.cloud.config.server.EnableConfigServer;

/**
 * spring cloud config server
 */
@EnableConfigServer
@SpringBootApplication
public class App 
{
    public static void main( String[] args )
    {
        SpringApplication.run(App.class, args);
    }
}

@EnableConfigServer注解用来开启配置服务功能。

5.启动服务，使用浏览器访问如下ws接口或使用curl:

http://localhost:8088/config/test

http://localhost:8088/config/test/master

http://localhost:8088/config-dev.yml

http://localhost:8088/master/config-dev.yml

http://localhost:8088/config-pro.properties

http://localhost:8088/master/config-pro.properties

均可看到对应的配置信息，说明服务成功获取到github上的配置。下面使用客户端应用来获取配置。

客户端：

1.新建spring boot应用，可参考前文

@SpringBootApplication
public class Application
{
    public static void main( String[] args )
    {
        SpringApplication.run(Application.class, args);
    }

}

2.添加配置文件application.properties：

server.port=8011 spring.application.name=config-client-name

3.添加配置文件bootstrap.properties：

spring.cloud.config.uri=http://localhost:8088/ spring.cloud.config.name=config spring.cloud.config.profile=dev spring.cloud.config.label=master

4.添加控制器类

@RestController
@RefreshScope
public class ConfigController {

    @Value("${id}")
    private String id;

    @Value("${desc}")
    private String desc;

    @Value("${db}")
    private String db;

    @RequestMapping("/test")
    public String test() {
        return this.toString();
    }

    @Override
    public String toString() {
        return "id:" + id + " desc:" + desc + " db:" + db;
    }
}

@RefreshScope用来开启配置刷新的触发动作，即更新对应字段的值

5.启动应用，curl http://localhost:8011/test 或使用浏览器访问，将看到对应的输出。

6.此时如果修改配置，再刷新页面，会发现输出不变，这是因为配置信息只在应用启动的时候加载一次，之后就不去重新加载了，

需要重新触发加载动作可以使用POST方式请求/refresh接口，即 curl -X POST http://localhost:8011/refresh

该接口位于spring-boot-starter-actuator依赖，调用前需添加否则404。

如遇报错如：{“timestamp”:1501254072321,“status”:401,“error”:“Unauthorized”,“message”:“Full authentication is required to access this resource.”,“path”:“/refresh”}

则需要在application配置文件中加入management.security.enabled=false或其他授权方式

参考 http://cloud.spring.io/spring-cloud-static/spring-cloud-config/1.3.1.RELEASE/

http://projects.spring.io/spring-cloud/spring-cloud.html
