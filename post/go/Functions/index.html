Go 函数

Posted @ 2018-09-09 · 2121 Words · Tags: Go · Hits:

基本函数示例:

// addCheckOverflow 完成2个int16相加,并且额外返回是否溢出
func addCheckOverflow(a, b uint16) (uint16, bool) {
    res := a + b
    overflowed := res < a || res < b
    return res, overflowed
}

func main() {
    res, overflowed := addCheckOverflow(1, 3)
    fmt.Printf("%5d + %5d = %5d, overflowed: %v\n\n", 1, 3, res, overflowed)
    res, overflowed = addCheckOverflow(65520, 10000)
    fmt.Printf("%5d + %5d = %5d, overflowed: %v\n", 65550, 10000, res, overflowed)
}

1 +     3 =     4, overflowed: false

65550 + 10000 = 9984, overflowed: true

参数

函数可以声明一些参数:

func SayHelloToMe(firstName, lastName string, age int) {
    fmt.Printf("Hello, %s %s!\n", firstName, lastName)
    fmt.Printf("You are %d\n", age)
}

func main() {
    SayHelloToMe("John", "Doe", 35)
}

Hello, John Doe! You are 35

省略firstName的类型，因为它与lastName相同。

返回值

函数可以返回一个或多个值给调用者:

func Add(a, b int) int {
    return a + b
}

func AddAndMultiply(a, b int) (int, int) {
    return a + b, a * b
}

func main() {
    sum, mult := AddAndMultiply(5, 8)
    fmt.Printf("5+8=%d, 5*8=%d\n", sum, mult)
    sum = Add(6, 12)
    fmt.Printf("6+12=%d\n", sum)
}

5+8=13, 5*8=40 6+12=18

命名返回值

可以给返回值命名，然后它们就像隐式局部变量一样。

之后可以使用空的return语句来返回它的值。 这就是所谓的“裸”返回。

func inverse(v float32) (reciprocal float32) {
    if v == 0 {
        return
    }
    reciprocal = 1 / v
    return
}

// 返回多个值
func split(sum int) (x, y int) {
    x = sum * 4 / 9
    y = sum - x
    return
}

func main() {
    fmt.Printf("inverse(5)=%.2f\n", inverse(5))
}

inverse(5)=0.20

需要注意的点:

  • 必须在返回值两边加上括号，
  • 必须要写return
  • 使用命名返回值被认为是不利于可读性的不良做法，尤其是在较长的函数中。 即显式胜于隐式。

可变参数函数

可变参数函数可以使用任意数量的参数。这些参数存储在slice中。

func variadic(strs ...string) {
    // strs is a slice of string
    for i, str := range strs {
        fmt.Printf("%d: %s\n", i, str)
    }
    fmt.Print("\n")
}

func main() {
    variadic("Hello", "Goodbye")
    variadic("Str1", "Str2", "Str3")

    // you can also give a slice to a variadic function, with `...`:
    strs := []string{"a", "b"}
    variadic(strs...)
}

0: Hello
1: Goodbye

0: Str1
1: Str2
2: Str3

0: a
1: b

匿名函数

匿名函数没有名字的函数

func main() {
    func() {
        fmt.Println("Hello!")
    }()
}

Hello!

带参数的函数表达式:

func main() {
    func(str string) {
        fmt.Println(str)
    }("Hello!")
}

Hello!

使用外部变量:

func main() {
    str := "Hello!"
    func() {
        fmt.Println(str)
    }()
}

Hello!

使用在函数外部定义的变量的函数称为闭包。

函数类型的变量

函数是Go中的一等值:

  • 函数可以分配给变量
  • 函数可以作为值传递给函数

将函数分配给变量:

// intOp is a variable whose type is function that takes
// 2 integers as arguments and returns an integer
var intOp func(int, int) int

func intAdd(a, b int) int {
    return a + b
}

func main() {
    intOp = intAdd
    fmt.Printf("intOp(2, 3) = %d\n", intOp(2, 3))

    // we can assign literal functions as well
    intOp = func(a, b int) int {
        return a * b
    }
    fmt.Printf("intOp(2, 3) = %d\n", intOp(2, 3))
}

intOp(2, 3) = 5
intOp(2, 3) = 6

函数作为参数传递

func funcAdd(a, b int) int {
    return a + b
}

func runFunc(a, b int, intOp func(int, int) int) {
    fmt.Printf("intOp(%d, %d) = %d\n", a, b, intOp(a, b))
}

func main() {
    runFunc(2, 3, funcAdd)

    // 也可以传递函数表达式
    runFunc(2, 3, func(a, b int) int {
        return a * b
    })
}

intOp(2, 3) = 5
intOp(2, 3) = 6

函数型参数的一般用法:

  • filepath.Walk会为每个找到的文件调用一个回调函数
  • ast.Inspect遍历树并为每个节点调用一个函数

函数的比较

函数变量只能和nil进行比较,

您无法将一个函数与另一个函数进行比较。 Go决定不执行此操作，因为用于比较函数的规则具有棘手的特殊情况，可能会让用户感到惊讶。

测试中的模拟功能

有时很难为一段代码编写测试。

假设您正在编写一个需要对用户进行身份验证的Web服务。

在生产部署中，这需要在数据库中查找用户信息。

在测试中，您不想与生产数据库对话。

启用调用用户身份验证的测试代码的一种方法是具有两种实现。

生产部署中使用的一种实现与数据库对话。

另一个实现正在伪造工作，并在测试中使用。

为了在它们之间切换，我们将变量用于一级间接。

这种技术称为mocking。

这是它可能如何工作的例子:

var isUserAdminFn func(string) bool

func isUserAdminProduction(userName string) bool {
    // an real implementation that talks to database
    return false
}

func isUserAdminMock(userName string) bool {
    // a fake implementation used for tests
    return userName == "admin"
}

func isUserAdmin(userName string) bool {
    return isUserAdminFn(userName)
}

func main() {
    isUserAdminFn = isUserAdminProduction
    // in test you would use:
    // isUserAdminFn = isUserAdminMock
}

闭包

闭包是与环境一起使用的功能。该函数通常是在另一个函数内部定义的匿名函数。环境是包围函数的词法范围(函数的词法范围的非常基本的概念就是存在于函数括号之间的范围)。

func g() {
    i := 0
    f := func() { // 匿名函数
        fmt.Println("f called")
    }
}

在一个函数(g)中定义的匿名函数(f)内，可以访问存在于f和g中的变量。

g的作用域构成了闭包的环境部分(功能部分为f)，对g作用域中的变量所做的更改将会保留(即环境在f的多次调用之间持续存在)。

通过例子说明:

func NaturalNumbers() func() int {
    i := 0
    f:= func() int { // f is the function part of closure
        i++
        return i
    }
    return f
}

在上面的定义中，NaturalNumbers具有一个内部函数f，NaturalNumbers返回该函数。 在f内部，正在访问在NaturalNumbers范围内定义的变量i。

我们从NaturalNumbers获得了一个新函数，如下所示:

n := NaturalNumbers()

现在n是一个闭包。它是一个函数(由f定义)，而且还具有关联的环境(NaturalNumbers范围)。

在n中，环境部分仅包含一个变量:i

由于n是一个函数，因此可以调用:

fmt.Println(n()) // 1
fmt.Println(n()) // 2
fmt.Println(n()) // 3

1
2
3

从上面的输出可以明显看出，每次调用n时，它都会递增i。 从0开始，对n的每个调用都执行i ++。

i的值在三次调用之间保留。 即作为闭包的一部分的环境持续存在。

再次调用NaturalNumbers()将创建并返回一个新函数。这将在NaturalNumbers中初始化一个新的i。
这意味着新返回的函数形成另一个闭包，该闭包具有与函数相同的部分(仍为f)，但具有全新的环境(新初始化的i)。

o := NaturalNumbers()

fmt.Println(n()) // 4
fmt.Println(o()) // 1
fmt.Println(o()) // 2
fmt.Println(n()) // 5

n和o都是包含相同功能部分(使它们具有相同行为)但环境不同的闭包。 因此，使用闭包允许函数访问持久性环境，该环境可用于保留调用之间的信息。

另一个例子:

func multiples(i int) func() int {
    var x int = 0
    return func() int {
        x++
        // paramenter to multiples (here it is i) also forms
        // a part of the environment, and is retained
        return x * i
    }
}

two := multiples(2)
fmt.Println(two(), two(), two()) // 2 4 6

fortyTwo := multiples(42)
fmt.Println(fortyTwo(), fortyTwo(), fortyTwo()) // 42 84 126

slice做函数参数

slice做函数参数,是值传递的,即在函数中操作的是值的拷贝,要想在函数中改变参数slice,则需要传入指针:

func testslice(ss *[]int){
	*ss = append(*ss, 4)
}
func main(){
    var s= []int{1,2,3}
	fmt.Println(s)
	testslice(&s)
	fmt.Println(s)
}
