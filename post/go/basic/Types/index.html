Go 基本数据类型

Posted @ 2018-09-09 · 2526 Words · Tags: Go · Hits:

bool

bool类型的值可以是true或false，初始为false

var b bool = true
fmt.Printf("b is: '%v'\n", b)
b = false
fmt.Printf("b is: '%v'\n", b)
var b2 bool
fmt.Printf("zero value of bool is: '%v'\n", b2)

b is: ‘true’
b is: ‘false’
zero value of bool is: ‘false’

bool变量的大小为1个byte。

b := true
fmt.Printf("size of bool is: %d\n", unsafe.Sizeof(b))

size of bool is: 1

int

Go有固定大小的无符号和有符号int类型:

  • int8, uint8
  • byte 是 uint8的别名
  • int16, uint16
  • int32, uint32
  • rune 是 int32的别名
  • int64, uint64

Go中还有与体系结构相关的整数:

int在32位处理器上为int32，在64位处理器上为int64 uint在32位处理器上是uint32，在64位处理器上是uint64

int的初始值为0。

用 strconv.Itoa 将int转化为string

var i1 int = -38
fmt.Printf("i1: %s\n", strconv.Itoa(i1))

var i2 int32 = 148
fmt.Printf("i2: %s\n", strconv.Itoa(int(i2)))

i1: -38
i2: 148

用 fmt.Sprintf将int转化为string

var i1 int = -38
s1 := fmt.Sprintf("%d", i1)
fmt.Printf("i1: %s\n", s1)

var i2 int32 = 148
s2 := fmt.Sprintf("%d", i2)
fmt.Printf("i2: %s\n", s2)

i1: -38
i2: 148

用 strconv.Atoi将string转为int

s := "-48"
i1, err := strconv.Atoi(s)
if err != nil {
    log.Fatalf("strconv.Atoi() failed with %s\n", err)
}
fmt.Printf("i1: %d\n", i1)

i1: -48

用 fmt.Sscanf将string转为int

s := "348"
var i int
_, err := fmt.Sscanf(s, "%d", &i)
if err != nil {
    log.Fatalf("fmt.Sscanf failed with '%s'\n", err)
}
fmt.Printf("i1: %d\n", i)

i1: 348

浮点类型

Go具有与IEEE 754标准相对应的浮点数类型:

  • float32 为 4 byte 长的浮点型数字 (即C语言中的float)
  • float64 为 8 byte 长的浮点型数字 (即C语言中的double)
  • float32 和 float64 的初始值为 0.0.

使用 FormatFloat 将float转为string

var f32 float32 = 1.3
bitSize := 32
s1 := strconv.FormatFloat(float64(f32), 'E', -1, bitSize)
fmt.Printf("f32: %s\n", s1)

var f64 float64 = 8.1234
bitSize = 64
s2 := strconv.FormatFloat(f64, 'e', -1, bitSize)
fmt.Printf("f64: %s\n", s2)

f32: 1.3E+00
f64: 8.1234e+00

用 Sprintf 将float转为字符串

var f64 float64 = 1.54
s := fmt.Sprintf("%f", f64)
fmt.Printf("f is: '%s'\n", s)

f is: ‘1.540000’

使用 strconv.FormatFloat 比 fmt.Sprintf 快

用 ParseFloat将string转为float

s := "1.2341"
f64, err := strconv.ParseFloat(s, 64)
if err != nil {
    log.Fatalf("strconv.ParseFloat() failed with '%s'\n", err)
}
fmt.Printf("f64: %f\n", f64)

f64: 1.234100

使用 Sscanf将string转为float

s := "1.2341"
var f float64
_, err := fmt.Sscanf(s, "%f", &f)
if err != nil {
    log.Fatalf("fmt.Sscanf failed with '%s'\n", err)
}
fmt.Printf("f: %f\n", f)

f: 1.234100

字符型

Go 有两种字符类型

  • byte是1个byte的值，是uint8类型的别名
  • rune是4个byte的Unicode值，是int32类型的别名 byte and rune的初始值为0

用byte遍历string

s := "str"
for i := 0; i < len(s); i++ {
    c := s[i]
    fmt.Printf("Byte at index %d is '%c' (0x%x)\n", i, c, c)
}

Byte at index 0 is ‘s’ (0x73)
Byte at index 1 is ‘t’ (0x74)
Byte at index 2 is ‘r’ (0x72)

用rune遍历string

s := "日本語"
for i, runeChar := range s {
    fmt.Printf("Rune at byte position %d is %#U\n", i, runeChar)
}

Rune at byte position 0 is U+65E5 ‘日’
Rune at byte position 3 is U+672C ‘本’
Rune at byte position 6 is U+8A9E ‘語’

string是byte组成的数组 当使用rune遍历string的使用go假定string是UTF-8编码的Unicode字符串 UTF-8是一种可变长度编码，可将rune编码为1、2、3或4个byte。 返回的索引i是tune开始的字符串中的字节位置。 而不是tune数量。

数组

Go中的数组具有固定大小。 因此，它们很少使用。在大多数情况下，我们使用可以扩大和缩小其大小的slice。数组基础使用方法:

var a1 = [2]byte{3, 8} // array of 2 bytes
// when using [...] size will be deduced from { ... }
a2 := [...]int{1, 2, 3} // array of 3 integers

fmt.Printf("Size of a1: %d.\nSize of a2: %d\n", len(a1), len(a2))

Size of a1: 2.
Size of a2: 3

切片

Go中的slice用于在大多数其他语言(如Ruby，Python或C＃)中使用数组，或在C ++中使用std :: vector。 切片的大小可以增长和缩小。 它被实现为对基础固定大小数组的视图。 切片的初始值值为nil。

map

map是从键到值的无序映射。其他语言称它们为字典(Python)或哈希表(C ++)。map的初始值为nil。 map基本使用方法:

m := make(map[string]int)
m["number3"] = 3

checkKey := func(k string) {
    if n, ok := m[k]; ok {
        fmt.Printf("value for key '%s' is %d\n", k, n)
    } else {
        fmt.Printf("key '%s' doesn't exist in map\n", k)
    }
}

// get value for a key that exists
checkKey("number3")

// get value for a key that doesn't exist
checkKey("number4")

// remove a key
delete(m, "number3")
fmt.Printf("deleted key 'number3\n")
checkKey("number3") // and now it doesn't exist

value for key ‘number3’ is 3
key ‘number4’ doesn’t exist in map
deleted key ‘number3
key ‘number3’ doesn’t exist in map

struct

结构将多个值分组为一个实体。零值结构是一个结构，其字段设置为各自的零值。

type MyStruct struct {
    IntVal           int
    StringVal        string
    unexportedIntVal int
}

Interface

接口在结构上定义了一组方法。接口的零值为nil。这是标准库中io.Reader接口的定义:

type Reader interface {
    Read(p []byte) (n int, err error)
}

指针

指向类型的指针是该类型的值在内存中的地址。指针的零值为nil。与C不同，Go没有指针运算, 可以输入变量的地址，但不能加减指针。

var a int = 4
pa := &a
fmt.Printf("Address of a variable in memory is %p. Its value is: %d\n", pa, *pa)

Address of a variable in memory is 0xc00007c010. Its value is: 4

Channel

通道是类型化队列，用于goroutine以线程安全的方式相互通信。通道的零值为零。 Channel使用示例

// create unbuffered channel of int values with capacity of 1
ch := make(chan int)
// start a new goroutine that sends value 3 over a channel
go func() { ch <- 3 }()
// read the value from a channel
// it waits until goroutine above sends a value
n := <-ch
fmt.Printf("n: %d\n", n)

n: 3

初始值

如果未显式分配Go中的变量，则使用已知值对其进行初始化,这个值称为零值. 这与C/C ++不同，在C/C ++中，未明确分配的变量具有未定义的值。

  • type zero value
  • bool false
  • integers 0
  • floating point numbers 0.0
  • string "”
  • pointer nil
  • slice nil
  • map nil
  • interface nil
  • channel nil
  • array all elements have zero value
  • struct all members set to zero value
  • function nil

通过代码查看:

类型转换

与C语言不同，Go不会进行类型之间的隐式转换,你必须在兼容类型之间显式转换:

// 在数字之间进行转换，即各种大小的整数和浮点数
var i1 int32 = 3
var i2 int = int(i1) // 从int32显式转换为int
var f float64 = float64(i1)

s := "string"
// we can cast between string and []byte and vice-versa
// note that unless optimizted by the compiler, this involves allocation
var d []byte = []byte(s)

类型别名

Go 1.9中引入了类型别名，以简化代码重构。

假设你有一个导出为类型Bar的包foo。

你想把类型Bar重命名为NewBar

如果没有类型别名，则必须在重命名类型的同时将所有使用foo.Bar的软件包更改为使用foo.NewBar。

使用类型别名分为两个步骤。

首先通过为foo.Bar引入别名并用别名替换foo.Bar的所有用法来更新依赖项。

import "foo"
type Bar = foo.Bar // Bar is now an alias of foo.Bar

现在将foo.Bar重命名为foo.NewBar并更新别名:

import "foo"
type Bar = foo.NewBar

这是一个非常小的改动.

然后逐渐摆脱别名并直接使用新的foo.NewBar类型。

这个过程听起来很麻烦，但是在大型代码库中，这比一次重命名所有内容更好。

在其他情况下使用类型别名很诱人，但是应该尽量不使用。

类型别名增加了一个间接层，这会损害代码的可读性。 应该有充分的理由使用它们。
