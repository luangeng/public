Go map类型

Posted @ 2019-09-09 · 2830 Words · Tags: Go · Hits:

map是无序的key-value对的集合.

map的零值为nil

map在其他语言中称为字典(C#, Python)或hash表(C++).

map上的迭代具有随机顺序,在这方面许多语言是不同的。

map基本用法:

m := make(map[string]int)

// set the value
m["three"] = 3
m["four"] = 4

// 获取值并查看值是否存在
key := "four"
if value, ok := m[key]; ok {
    fmt.Printf("Key '%s' exists and maps to %d\n", key, value)
} else {
    fmt.Printf("Key '%s' doesn't exists\n", key)
}

key = "five"
if value, ok := m[key]; ok {
    fmt.Printf("Key '%s' exists and maps to %d\n", key, value)
} else {
    fmt.Printf("Key '%s' doesn't exists\n", key)
}

// if value doesn't exist, the result of lookup is zero value. In this case zero value of int is 0
fmt.Printf("\nValue for non-existing key: %d\n\n", m["not-exists"])

// iterating over keys and values
fmt.Printf("All keys and their values:\n")
for key, value := range m {
    fmt.Printf("%s => %d\n", key, value)
}

fmt.Printf("\nBefore deletion: len(m)=%d\n", len(m))
delete(m, "four")
fmt.Printf("After deletion : len(m)=%d\n", len(m))

Key ‘four’ exists and maps to 4
Key ‘five’ doesn’t exists

Value for non-existing key: 0

All keys and their values:
three => 3
four => 4

Before deletion: len(m)=2
After deletion : len(m)=1

map的声明和初始化

使用map关键字声明map, 后面是key和value的类型:

// Keys are ints, values are ints.
var m1 map[int]int // initialized to nil

// Keys are strings, values are ints.
var m2 map[string]int // initialized to nil

map是引用类型，一旦定义，它们的零值为nil。 写入nil map将导致panic，而读取将始终返回其零值。

使用make方法初始化map

m := make(map[string]int)

使用make的两参数形式，可以为map指定初始输入容量，而不是默认容量：

m := make(map[string]int, 30)

另外，可以声明一个映射，将其初始化为零值，然后再为它分配一个文字值，如果将该结构序列化为json，从而在返回时生成一个空映射，则将有帮助。

m := make(map[string]int, 0)

也可以使用{}初始化map的值

var m map[string]int = map[string]int{"Foo": 20, "Bar": 30}

fmt.Println(m["Foo"]) // outputs 20

以下所有语句导致该变量绑定到相同的值。

// Declare, initializing to zero value, then assign a literal value.
var m map[string]int
m = map[string]int{}

// Declare and initialize via literal value.
var m = map[string]int{}

// Declare via short variable declaration and initialize with a literal value.
m := map[string]int{}

我们还可以使用map文字来创建带有一些初始键/值对的新map。

key类型可以是任何可比较的类型。值得注意的是，这不包括函数，映射和切片。 值类型可以是任何类型，包括自定义类型或接口{}。

type Person struct {
    FirstName string
    LastName  string
}

// Declare via short variable declaration and initialize with make.
m := make(map[string]Person)

// Declare, initializing to zero value, then assign a literal value.
var m map[string]Person
m = map[string]Person{}

// Declare and initialize via literal value.
var m = map[string]Person{}

// Declare via short variable declaration and initialize with a literal value.
m := map[string]Person{}

创建map

可以使用复合语法在单个语句中声明和初始化map。

使用自动类型短变量声明：

mapIntInt := map[int]int{10: 100, 20: 100, 30: 1000}
mapIntString := map[int]string{10: "foo", 20: "bar", 30: "baz"}
mapStringInt := map[string]int{"foo": 10, "bar": 20, "baz": 30}
mapStringString := map[string]string{"foo": "one", "bar": "two", "baz": "three"}

或者

var mapIntInt = map[int]int{10: 100, 20: 100, 30: 1000}
var mapIntString = map[int]string{10: "foo", 20: "bar", 30: "baz"}
var mapStringInt = map[string]int{"foo": 10, "bar": 20, "baz": 30}
var mapStringString = map[string]string{"foo": "one", "bar": "two", "baz": "three"}

还可以在map中包含自己的结构,即使用自定义类型作为值：

// Custom struct types
type Person struct {
  FirstName, LastName string
}

var mapStringPerson = map[string]Person{
  "john": Person{"John", "Doe"},
  "jane": Person{"Jane", "Doe"}}
mapStringPerson := map[string]Person{
  "john": Person{"John", "Doe"},
  "jane": Person{"Jane", "Doe"}}

struct也可以作为map的key:

type RouteHit struct {
    Domain string
    Route  string
}

var hitMap = map[RouteHit]int{
    RouteHit{"example.com","/home"}: 1,
    RouteHit{"example.com","/help"}: 2}
hitMap := map[RouteHit]int{
    RouteHit{"example.com","/home"}: 1,
    RouteHit{"example.com","/help"}: 2}

只需在方括号{}中不输入任何值即可创建一个空的map。

mapIntInt := map[int]int{}
mapIntString := map[int]string{}
mapStringInt := map[string]int{}
mapStringString := map[string]string{}
mapStringPerson := map[string]Person{}

可以直接创建和使用map，而无需将其分配给变量。 但是，你必须同时指定声明和内容。

// map最为 fmt.Println()的参数
fmt.Println(map[string]string{
  "FirstName": "John",
  "LastName": "Doe",
  "Age": "30"})

// 等效于
data := map[string]string{
    "FirstName": "John",
    "LastName": "Doe",
    "Age": "30"}
fmt.Println(data)

通过key获取value

要获取map中的value,只需要value := mapName[ key ]

如果map中包含这个key, 则会返回对应的value;如果不包含这个key,则会返回value的类型的零值(int的话是0,string的话是”")

m  := map[string]string{"foo": "foo_value", "bar": ""}
k  := m["foo"]  // returns "foo_value" since that is the value stored in the map
k2 := m["bar"]  // returns "" since that is the value stored in the map
k3 := m["nop"]  // returns "" since the key does not exist, and "" is the string type's zero value

那么要区分零值和不存在的键，可以使用get的第二个返回值(例如value, hasKey：= map [“ key”])

第二个值是布尔型

  • true 当map中包含这个key
  • false map中不包含这个key

示例:

value, hasKey = m[key]
if hasKey {
    // the map contains the given key, so we can safely use the value
    // If value is zero-value, it's because the zero-value was pushed to the map
} else {
    // The map does not have the given key
    // the value will be the zero-value of the map's type
}

获取map的大小 (map中元素的数量)

内部函数len返回了map中元素的数量:

m := map[string]int{}
size := len(m)
fmt.Printf("Size of empty map: %d\n", size)

m["foo"] = 1
size = len(m)
fmt.Printf("Size of map: %d\n\n", size)

Size of empty map: 0 Size of map: 1

nil map的大小是0:

var m map[string]int
size := len(m)
fmt.Printf("Size of nil map: %d\n\n", size)

Size of nil map: 0

拷贝map

在Go中，所有值都通过拷贝传递。

然而,和slice类似,映射的值仅是对底层数据的引用。当将map分配给另一个变量或传递给另一个函数时，仅复制引用。

要想拷贝值,需要写下面一段代码:

src := make(map[string]int)
src["one"] = 1
src["two"] = 2

dst := make(map[string]int)

for key, value := range src {
    dst[key] = value
}

使用range遍历map

未指定迭代顺序。 故意随机化迭代顺序，以使代码不会错误地依赖于特定顺序。

同时迭代key和value:

people := map[string]int{
    "john": 30,
    "jane": 29,
    "mark": 11,
}

for key, value := range people {
    fmt.Printf("key: %s, value: %d\n", key, value)
}

key: john, value: 30
key: jane, value: 29
key: mark, value: 11

仅迭代key

people := map[string]int{
    "john": 30,
    "jane": 29,
    "mark": 11,
}

for key := range people {
    fmt.Printf("key: %s\n", key)
}

key: mark
key: john
key: jane

仅迭代value

people := map[string]int{
    "john": 30,
    "jane": 29,
    "mark": 11,
}

for _, value := range people {
    fmt.Printf("value: %d\n", value)
}

value: 30
value: 29
value: 11

从map中删除元素

使用内部函数delete从map中删除指定key的元素:

people := map[string]int{"john": 30, "jane": 29}
fmt.Printf("%v\n", people)

delete(people, "john")
fmt.Printf("%v\n", people)

map[jane:29 john:30]
map[jane:29]

如果map 是nil或者map中没有这个key, 则删除操作无效:

people := map[string]int{"john": 30, "jane": 29}
fmt.Printf("%v\n", people)

// deleting a key that doesn't exist is a no-op
delete(people, "notfound")
fmt.Printf("%v\n", people)

// deleting a key in nil map is a no-op
var something map[string]int
delete(something, "notfound")

map[jane:29 john:30]
map[jane:29 john:30]

把map作为set使用

一些语言提供了内置的set类型

在Go中，最佳实践是使用值类型为struct{}的map，因为值为空struct的map进行了特别优化。

例如一个string类型的set:

set := NewStringSet("hi", "hello")
fmt.Printf("Original set: %#v\n", set.Strings())

set.Delete("hi")
fmt.Printf("After delete: %#v\n", set.Strings())

set.Add("hey")
fmt.Printf("After add   : %#v\n", set.Strings())

printExists(set, "hello")
printExists(set, "ola")

Original set: []string{“hi”, “hello”}
After delete: []string{“hello”}
After add : []string{“hey”, “hello”}
‘hello’ exists in set
‘ola’ doesn’t exist in set

slice作为map的value

m := make(map[string][]int)
fmt.Printf("Non-existing key returns zero value of []int, i.e. nil: %#v\n", m["key1"])

m["key1"] = append(m["key1"], 8)
m["key1"] = append(m["key1"], 3)
fmt.Printf("\nAfter appending to slice: %#v\n", m["key1"])

delete(m, "key1")
fmt.Printf("\ndelete() sets back to nil slice: %#v\n", m["key1"])

Non-existing key returns zero value of []int, i.e. nil: []int(nil)
After appending to slice: []int{8, 3}
delete() sets back to nil slice: []int(nil)

注意到在m[“key1”] = append(m[“key1”], 8)时我们没有检查m[“key1”]是否存在. 这是三个规则相互影响的结果：

  1. m[“foo”] 如果不存在foo键，则返回零值
  2. slice的零值是nil
  3. a = append(nil, 3)语句合法并且创建了一个单元素的slice[]int{3}

并发访问map

Go中的map不是并发安全的

确保并发安全的一种访问方法是使用sync.Mutex或sync.RWMutex锁定所有映射进行读取和写入。

例如，一个简单的map包装器，只要通过Get，Set和Add方法完成，即可安全地并发使用：

package main

import (
    "fmt"
    "sync"
)

// ConcurrentMap is a wrapper around map that is safe for concurrent use
type ConcurrentMap struct {
    sync.RWMutex
    m map[string]int
}

// Get gets a value for a key
func (r *ConcurrentMap) Get(key string) int {
    r.RLock()
    defer r.RUnlock()
    return r.m[key]
}

// Set sets a key to a given value
func (r *ConcurrentMap) Set(key string, val int) {
    r.Lock()
    defer r.Unlock()
    r.m[key] = val
}

// Add increases the value under a key by n
func (r *ConcurrentMap) Add(key string, n int) int {
    r.Lock()
    defer r.Unlock()
    r.m[key] += n
    return r.m[key]
}

func main() {
    counter := ConcurrentMap{m: make(map[string]int)}

    n := counter.Add("some_key", 2)
    fmt.Printf("Final value is: %d\n", n)
}

Final value is: 2

sync.RWMutex can be faster because it allows multiple readers. On the downside you have to be more careful to ensure you’re not writing under RLock. The performance difference with sync.Mutex will be negligible unless there’s a high-contention on the lock or the locks are held for a long time.

sync.RWMutex可以更快，因为它允许多个读取器。 不利的一面是，你必须更加小心以确保不在RLock下书写。 sync.Mutex的性能差异可以忽略不计，除非锁上存在高争用或锁被长时间持有。

map的零值

map的零值是nil. 它的长度为0:

var m map[string]string
fmt.Printf("m == nil ? %v\n", m == nil)
fmt.Printf("len(m) = %d\n", len(m))

m == nil ? true
len(m) = 0

从nil map读取和从空map读取一样。 而写入nil map则会报错。

var m map[string]string

// 可以从未初始化的map中读
fmt.Printf(`m["foo"] = %s`+"\n", m["foo"])
_, ok := m["foo"]
fmt.Printf("ok: %v\n", ok)

// 不能写入未初始化的map
m["foo"] = "bar"

m[“foo”] =
ok: false
panic: assignment to entry in nil map
