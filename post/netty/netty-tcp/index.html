Netty--TCP协议发送和接收数据

Posted @ 2017-09-13 · 1272 Words · Tags: netty · Hits:

Netty是业界最流行的NIO开发框架，使用Netty可以构建高性能异步通信能力，经常用于RPC底层通信框架。Netty的API使用简单，功能强大，预置了多种编解码功能，支持多种主流协议，并且可以通过ChannelHandler对通信框架进行灵活的扩展；Netty并发量高，性能好，成熟稳定，是Java NIO开发的首选框架。

以下是一个使用Netty通信在TCP协议的应用

服务端TcpServer.java

public class TcpServer {
    private static final int PORT = 9000;
    private static Logger logger = LoggerFactory.getLogger(TcpServer.class);

    protected static void start() {
        final EventLoopGroup bossGroup = new NioEventLoopGroup(2);
        final EventLoopGroup workerGroup = new NioEventLoopGroup(4);
        try {
            ServerBootstrap b = new ServerBootstrap();
            b.group(bossGroup, workerGroup);
            b.channel(NioServerSocketChannel.class);
            b.childHandler(new ChannelInitializer<SocketChannel>() {
                @Override
                public void initChannel(SocketChannel ch) throws Exception {
                    ChannelPipeline pipeline = ch.pipeline();
                    pipeline.addLast("frameDecoder", new LengthFieldBasedFrameDecoder(Integer.MAX_VALUE, 0, 4, 0, 4));
                    pipeline.addLast("frameEncoder", new LengthFieldPrepender(4));
                    pipeline.addLast("decoder", new StringDecoder(CharsetUtil.UTF_8));
                    pipeline.addLast("encoder", new StringEncoder(CharsetUtil.UTF_8));
                    pipeline.addLast(new TcpServerHandler());
                }
            });

            // Start the server.
            ChannelFuture f = b.bind(PORT).sync();
            logger.info("TCP服务器已启动");

            // Wait until the server socket is closed.
            f.channel().closeFuture().sync();
        } catch (InterruptedException e) {
            e.printStackTrace();
        } finally {
            workerGroup.shutdownGracefully();
            bossGroup.shutdownGracefully();
        }
    }

    public static void main(String[] args) throws Exception {
        TcpServer.start();
    }
}

NioEventLoopGroup是线程组，包含一组NIO线程，NIO线程用于网络事件的处理，等同于JDK Reactor线程组。bossGroup用于服务端接收客户端的连接，workerGroup用于进行SocketChannel的网络读写。

ServerBootstrap是Netty用于启动NIO服务端的辅助启动类，用它简化服务端的开发。

b.group方法用于初始化线程组。

NioServerSocketChannel对应于JDK中的ServerSocketChannel。

b.ChildHandler用于绑定事件处理，如IO事件，编解码等。

服务端自定义Handler ，用于处理接收到的网络事件

public class TcpServerHandler extends SimpleChannelInboundHandler<Object> {

    private static Logger logger = LoggerFactory.getLogger(TcpClient.class);

    @Override
    protected void channelRead0(ChannelHandlerContext ctx, Object msg) throws Exception {
        ctx.channel().writeAndFlush("server收到消息:" + msg);
    }

    @Override
    public void exceptionCaught(ChannelHandlerContext ctx, Throwable cause) throws Exception {
        logger.warn("exceptionCaught!", cause);
        ctx.close();
    }

}

channelRead方法在收到消息后触发，Netty的write方法只把待发送消息放在缓冲中，flush方法将缓冲中的方法写到SocketChannel中。

exceptionCaught方法在发生异常的时候调用。

客户端TcpClient.java

public class TcpClient {
    public static String HOST = "127.0.0.1";
    public static int PORT = 9000;
    private static Logger logger = LoggerFactory.getLogger(TcpClient.class);

    public static final void send() {
        Bootstrap bootstrap = new Bootstrap();
        EventLoopGroup group = new NioEventLoopGroup();
        try {
            bootstrap.group(group).channel(NioSocketChannel.class);
            bootstrap.handler(new ChannelInitializer<Channel>() {
                @Override
                protected void initChannel(Channel ch) throws Exception {
                    ChannelPipeline pipeline = ch.pipeline();
                    pipeline.addLast("frameDecoder", new LengthFieldBasedFrameDecoder(Integer.MAX_VALUE, 0, 4, 0, 4));
                    pipeline.addLast("frameEncoder", new LengthFieldPrepender(4));
                    pipeline.addLast("decoder", new StringDecoder(CharsetUtil.UTF_8));
                    pipeline.addLast("encoder", new StringEncoder(CharsetUtil.UTF_8));
                    pipeline.addLast("handler", new TcpClientHandler());
                    //pipeline.addLast("log", new LoggingHandler(LogLevel.INFO));
                }
            });
            bootstrap.option(ChannelOption.SO_KEEPALIVE, true);

            // Start the client.
            ChannelFuture f = bootstrap.connect(HOST, PORT).sync();
            logger.info("连接建立!");

            for (int i = 0; i < 60; i++) {
                f.channel().writeAndFlush("hello" + i).sync();
                TimeUnit.SECONDS.sleep(1);
            }

            // Wait until the connection is closed.
            f.channel().closeFuture().sync();
        } catch (InterruptedException e) {
            e.printStackTrace();
        } finally {
            group.shutdownGracefully();
        }
    }

    public static void main(String[] args) throws Exception {
        TcpClient.send();
    }

}

NioEventLoopGroup为事件处理线程组。

客户端的Channel设置为NioSocketChannel，ChannelPipeline为事件处理链。

调用connect方法发起异步连接，调用sync同步等待连接成功。

循环调用channel.writeAndFlush发送消息。

客户端自定义Handler，处理服务端返回的消息

public class TcpClientHandler extends SimpleChannelInboundHandler<Object> {
    private static Logger logger = LoggerFactory.getLogger(TcpClient.class);

    @Override
    protected void channelRead0(ChannelHandlerContext ctx, Object msg) throws Exception {
        logger.info("client接收到服务器返回的消息:" + msg);
    }

}
